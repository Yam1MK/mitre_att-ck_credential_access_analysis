# 1. Выбранная тактика

Название тактики: ***Credential Access*** 
Матрица: Linux (attack.mitre.org/matrices/enterprise/linux/)


# 2. Подробное описание тактики

- *Цель и место в цепочке поставки:*

    злоумышленник пытается украсть имена пользователей и пароли от учетных записей. К методам получения учетных данных относятся перехват нажатий клавиш или дампа учетных данных. Использование легитимных учетных данных может предоставить злоумышленникам доступ к системам, затруднить их обнаружение и дать возможность создать больше учетных записей для достижения своих целей.


- *Описание техник, связанных с данной тактикой:*

    1)  ID - ***T1557***

        Имя - Adversary-in-the-Middle

        Описание - злоумышленники могут попытаться занять позицию между двумя или более      сетевыми устройствами, чтобы обеспечить последующие действия, такие как перехват сетевого трафика , манипулирование передаваемыми данными или атаки повторного воспроизведения (эксплуатация для получения доступа к учетным данным).

    2)  ID - ***T1110***

        Имя - Brute Force

        Описание - злоумышленники могут использовать методы перебора для получения доступа к учетным записям, когда пароли неизвестны или когда получены хеши паролей.  Перебор паролей может происходить посредством взаимодействия со службой, которая проверяет достоверность этих учетных данных, или в автономном режиме с использованием ранее полученных данных учетных данных.

    3) ID - ***T1555***

        Имя - Credentials from Password Stores

        Описание - злоумышленники могут искать распространенные места хранения паролей, чтобы получить учетные данные пользователей. Например, существуют специальные приложения и сервисы (менеджеры паролей, облачные хранилища секретов), которые хранят пароли, чтобы упростить их управление и обслуживание пользователями.

    4)  ID - ***T1212***

        Имя - Exploitation for Credential Access

        Описание - злоумышленники могут использовать уязвимости программного обеспечения в попытке получить учетные данные. Эксплуатация уязвимости программного обеспечения происходит, когда злоумышленник использует ошибку в работе программы, службы или в самом программном обеспечении операционной системы или её ядре.

    5)  ID - ***T1187***

        Имя - Forced Authentication

        Описание - злоумышленники могут собирать учетные данные, заставляя пользователя автоматически предоставлять информацию для аутентификации с помощью механизма, который они могут перехватить.

    6)  ID - ***Т1606***

        Имя - Forge Web Credentials

        Описание - злоумышленники могут подделывать учетные данные, которые можно использовать для получения доступа к веб-приложениям или интернет-сервисам. Веб-приложения и сервисы часто используют сессионные файлы cookie, токены или другие материалы для аутентификации и авторизации доступа пользователей.

    7)  ID - ***Т1056***

        Имя - Input Capture

        Описание - злоумышленники могут использовать методы перехвата пользовательского ввода для получения учетных данных или сбора информации. В процессе обычной работы системы пользователи часто предоставляют учетные данные для различных мест, таких как страницы входа/порталы или диалоговые окна системы. Механизмы перехвата ввода могут быть основаны на обмане пользователя с целью получения ввода, который он считает подлинным сервисом.

    8)  ID - ***T1556***

        Имя - Modify Authentication Process

        Описание - злоумышленники могут изменять механизмы и процессы аутентификации для доступа к учетным данным пользователей или предоставления иного несанкционированного доступа к учетным записям. Изменяя процесс аутентификации, хакер может получить возможность аутентифицироваться в службе или системе без использования действительных учетных записей.

    9)  ID - ***T1111***

        Имя - Multi-Factor Authentication Interception

        Описание - злоумышленники могут использовать механизмы мультифакторной аутентификации (смарт-карты, генераторы токенов и т. д.) для получения доступа к учетным данным, которые могут быть использованы для доступа к системам, сервисам и сетевым ресурсам. Следует знать о методах, которые могут быть использованы для перехвата и обхода этих механизмов безопасности.

    10) ID - ***T1621***

        Имя - Multi-Factor Authentication Request Generation

        Описание - злоумышленники могут попытаться обойти механизмы мультифакторной аутентификации (МФА) и получить доступ к учетным записям, генерируя запросы на МФА и отправляя их пользователям.

    11) ID - ***T1040***

        Имя - Network Sniffing

        Описание - злоумышленники могут перехватывать сетевой трафик для получения информации об окружающей среде, включая аутентификационные данные, передаваемые по сети. Злоумышленник может перевести сетевой интерфейс в promiscuous mode (принятие всего проходящего трафика) для пассивного доступа к данным, передаваемым по сети, или использовать порты SPAN для перехвата большего объема данных.

    12) ID - ***T1003***

        Имя - OS Credential Dumping

        Описание - злоумышленники могут попытаться получить доступ к учетным данным, таким как логин и пароль, обычно в виде хеша или открытого текста. Учетные данные могут быть получены из кэша операционной системы, памяти или структур. Затем эти данные могут быть использованы для осуществления горизонтального перемещения и доступа к ограниченной информации.

    13) ID - ***T1528***

        Имя - Steal Application Access Token

        Описание - злоумышленники могут красть токены доступа к приложениям, чтобы получить учетные данные для доступа к удаленным системам и ресурсам.

    14) ID - ***T1649***

        Имя - Steal or Forge Authentication Certificates

        Описание - злоумышленники могут украсть или подделать сертификаты, используемые для аутентификации, чтобы получить доступ к удаленным системам или ресурсам. Цифровые сертификаты часто используются для подписи и шифрования сообщений и/или файлов.

    15) ID - ***Т1558***

        Имя - Steal or Forge Kerberos Tickets

        Описание - злоумышленники могут попытаться обойти аутентификацию, украв или подделав билеты Kerberos, чтобы активировать механизм «Передачи билета» (Pass the Ticket).Kerberos — это протокол аутентификации, широко используемый в современных доменных средах Windows. Злоумышленники могут попытаться злоупотребить Kerberos, украв билеты или подделав билеты для обеспечения несанкционированного доступа.

    16) ID - ***T1539***

        Имя - Steal Web Session Cookie

        Описание - злоумышленник может украсть сессионные файлы cookie веб-приложений или сервисов и использовать их для получения доступа к веб-приложениям или интернет-сервисам в качестве авторизованного пользователя без необходимости ввода учетных данных.

    17) ID - ***Т1552***

        Имя - Unsecured Credentials

        Описание - злоумышленники могут обыскивать скомпрометированные системы в поисках и получении небезопасно хранящихся учетных данных. Эти учетные данные могут храниться и/или быть потеряны во многих местах системы, включая текстовые файлы (например, история командной оболочки), хранилища операционной системы или приложений (например, учетные данные в реестре) или другие специализированные файлы/артефакты (например, закрытые ключи).


- *Примеры реальных атак, где использовалась данная тактика*

    - ID T1557 (Adversary-in-the-Middle)

        **ArcaneDoor** — это кампания, направленная на сетевые устройства Cisco и других производителей в период с июля 2023 года по апрель 2024 года, в основном ориентированная на правительственные сети и сети критической инфраструктуры. Она связана с развертыванием пользовательских бэкдоров Line Runner и Line Dancer. ArcaneDoor приписывается группе, известной как UAT4356 (так именуют в Talos) или STORM-1849 (так именуют в Microsoft), и считается кампанией, спонсируемой государством.

        ArcaneDoor подключал перехват HTTP-трафика к устройствам жертв для идентификации и анализа информации управления, отправляемой на устройство.

    - ID T1110 (Brute Force)

        **APT38** - это спонсируемая Северной Кореей группа киберпреступников, специализирующаяся на финансовых кибероперациях. Её связывают с Главным разведывательным управлением. Данная группа преступников, активная с 2014 года, атаковала банки, финансовые учреждения, казино, криптовалютные биржи, конечные точки системы SWIFT и банкоматы как минимум в 38 странах мира.

        APT38 использует методы перебора для попыток доступа к учетным записям, когда пароли неизвестны или когда хеши паролей недоступны.

    - ID T1556 (Modify Authentication Process)

        **Ebury** - это бэкдор OpenSSH и программа для кражи учетных данных, разработанная компанией Windigo и нацеленная на серверы Linux и хосты контейнеров. Ebury устанавливается, в основном, путём модификации разделяемых библиотек (.so файлов), выполняемых легитимной программой OpenSSH. Впервые обнаруженный в 2009 году Ebury использовался для поддержания ботнета (заражения сети устройств вредоносным ПО) серверов, развертывания дополнительного вредоносного ПО и кражи криптовалютных кошельков, учетных данных и данных кредитных карт.

        Ebury способен перехватывать закрытые ключи с помощью троянской ssh-add функции, тем самым получая возможность расшифровывать ssh трафик.

    - ID T1539 (Steal Web Session Cookie)

        **DarkGate** - вредоносное ПО, появившееся в 2018 году и являющееся инструментом для первоначального доступа и сбора данных, связанным с различными преступными кибер-операциями. Написанный на Delphi DarkGate ассоциируется с кражей учетных данных, криптомайнингом, криптокражей и вымогательством. Его использование значительно возросло, начиная с 2022 года, и он находится в активной разработке своим автором, который предоставляет его как услугу «вредоносное ПО как сервис» (Malware-as-a-Service).

        DarkGate способен украсть файлы cookie Opera, если таковые имеются, после завершения соответствующего процесса.


# 3. Киберпреступные группировки, которые использовали/используют данную тактику

Как и писалось ранее, данную тактику использовали такие преступные группировки как *ArcaneDoor*, *APT38*. Также стоит упомянуть довольно известные объединения как *APT33* (группа из Ирана), *DarkHydrus* (группа хакеров, атакующая правительственные учреждения на Ближнем Востоке), *Chimera* (предполагаемая группировка из Китая), *APT42* (спонсируемая Ираном группа киберпреступников) и др.

Использование данной тактики проявляется в реальных инцидентах следующим образом:

- возможность с помощью поддельных сайтов обманывать пользователей, заставляя их вводить свои учётные данные;

- зарегестрировать альтернативные телефонные номера для взломанных пользователей, чтобы перехватывать коды двухфакторной аутентификации;

- злоупотребление неправильно настроенными шаблонами сертификатов AD CS для выдачи себя за администраторов и создания дополнительных сертификатов аутентификации;

- получение учётных данных пользователей, небезопасно хранящиеся на целевых сетевых устройствах и т.д.

Это лишь небольшое количество реальных примеров использования и эксплуатации данной тактики, которыми могут воспользоваться злоумышленники и украсть учётные данные обычных пользователей.


# 4. Практическая часть

В ходе выполнения практической части была произведена тактика *Credential Access* (доступ к учётным данным), а также техника Brute Force (перебор) для попытки прохождения аутентификации и получения прав администратора на такой платформе как OWASP Juice Shop.

Сперва были применены навыки осинта, чтобы на самом сайте найти почту администратора. Далее нужно сымитировать перебор пароля от логина администратора. Для этого был написан небольшой скрипт на языке Python, в котором была использована библиотека requests для установки соединения и отправки POST-запросов на сервер. В коде также используется простой цикл, который перебирает файл с 100 тыс. самых популярных паролей и подставляет их в POST-запрос, и, если status code сервера равен 200, то на экран выводится сообщение об успешном нахождении валидного пароля.


#  5. Разработка и обоснование методов защиты от атак, реализующих данную тактику

- *Наиболее эффективные методы защиты от данных атак*

    В случае с данной тактикой наиболее важными методами защиты, на которых стоит сфокусироваться, являются мониторинг, ограничение взаимодействия пользователей с ресурсом по мере необходимости, корректная обработка ошибок, а также проверка аутентификации пользователей.

    Под ограничением подразумевается, например, фиксированное количество запросов от одного пользователя за небольшой промежуток времени. Это требуется для борьбы с Dos-атаками и Brute Force.

    С помощью мониторинга возможно обнаружение различных атак на раннем этапе, чтобы была возможность заранее их отразить. Также с помощью него можно выявить вектор и технику атаки.

    Корректная обработка ошибок подразумевает экранизацию входных и выходных данных сервера, дабы предотвратить различные иньекции со стороны вредоностного пользователя.

    Проверка аутентификации лишает злоумышленника возможности получения прав авторизованных пользователей и не даёт ему доступа к конфиденциальным данным.

- *Инструменты обнаружения и реагирования*

    В первую очередь для обнаружения самих аномалий потребуется система SIEM (Security Information and Event Managment), которая используется для сбора логов систем активной защиты и их структурирования для последующего удобного анализа и построения плана отражения атаки.

    Также для отслеживания угроз можно использовать такой инструмент как EDR (Endpoint Detection and Response), который мониторит эндпоинты на наличие уязвимостей и различных инъекций и выполняет автоматические действия для их устранения.

    Для выявления аномалий можно воспользоваться инструментом UEBA (User and Enitity Behavior Analytics), который отслеживает нестандартную активность пользователей, такую как необычное время активности, локация и т.д.


- *Best practices по настройке систем и политик*

    Лучшими способами по настройке систем и политик безопасности можно считать:

    1) Защита процессов аутентификации;

    2) Политики аутентификации;

    3) Управление привилегиями;

    4) Ограничение доступа к файловой системе и реестру;

    5) Многофакторность.

    Эти способы должны применяться для обеспечения надёжности и полноценной безопасности сервисов в информационной среде.
